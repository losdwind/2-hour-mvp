# 产品视频流水线 · 统一入口

四个阶段，四个交付物。本技能负责判断起点、按序调度阶段技能、在阶段间设检查点。

| 阶段 | 技能 | 输入 | 交付物 |
|------|------|------|--------|
| 1 | idea-to-prd | 一个点子 | PRD markdown（含页面清单与用户旅程） |
| 2 | prd-to-hifi | PRD | 单文件高保真交互稿 HTML（可点击） |
| 3 | hifi-to-storyboard | HTML 或整套截图 | 页面截图 + 分镜表 + seedream 分镜图 |
| 4 | storyboard-to-video | 分镜表 + 分镜图 | 视频成片 mp4 + 质检抽帧图 |

## 第 1 步 · 判断起点

看用户给了什么，从最靠后的阶段进入，别把用户已有的产物重做一遍：

- 只有一句话点子 → 阶段 1 起步
- 有 PRD（或等价的需求描述文档）→ 阶段 2 起步
- 有多页面 HTML 交互稿或整套页面截图 → 阶段 3 起步
- 有分镜表和标准帧图 → 阶段 4 起步

拿不准时问用户一次。

## 第 2 步 · 逐阶段执行

每个阶段读对应技能文件并严格照做：

```
${CLAUDE_PLUGIN_ROOT}/skills/idea-to-prd/SKILL.md
${CLAUDE_PLUGIN_ROOT}/skills/prd-to-hifi/SKILL.md
${CLAUDE_PLUGIN_ROOT}/skills/hifi-to-storyboard/SKILL.md
${CLAUDE_PLUGIN_ROOT}/skills/storyboard-to-video/SKILL.md
```

## 第 3 步 · 阶段间检查点

每个阶段交付后停下来让用户确认再进下一阶段，理由有三：

1. **上游错误会被下游放大**。PRD 页面清单漏一页，交互稿就缺一页，视频就少一镜；返工成本逐级上涨。
2. **阶段 3 起要花即梦积分**。进入阶段 3 前必须报积分预估并拿到用户确认；阶段 4 是最大支出，试跑通过后才批量。
3. **交付物本身有独立价值**。用户可能只想要 PRD 或交互稿，别自作主张一路跑到底。

用户明确说"一路跑到底不用确认"时，检查点降级为进度汇报，但积分确认不可省。

## 全程纪律

- 交付物命名沿用各阶段技能的约定，页面 id 从 PRD 一路贯穿到分镜帧文件名。
- 每阶段结束报一句进度（阶段几、交付物路径、下一步）。
- 中断恢复时先盘点已有产物落在哪个阶段，从断点续跑，tasklog/ 里有 submit_id 的生成任务先免费取回。

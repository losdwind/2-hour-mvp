# PRD → 高保真交互稿

产出一个单文件 HTML，左侧页面导航 + 中间设备框逐页展示 + 每页交互说明。它同时服务三种消费者：人（点着看）、评审（读交互说明）、hifi-to-storyboard 技能（逐页截图做演示视频），所以 DOM 结构约定必须严格遵守。

## 第 1 步 · 输入与问答

有 PRD 就读 PRD 第 5 节页面清单和第 6 节用户旅程；没有 PRD 只有想法时，先花两轮问答把页面清单问出来（也可以建议用户先跑 idea-to-prd）。

用 AskUserQuestion 确认视觉盘子（一次问完）：

1. **风格气质**。给 3-4 个候选（如 奶油粉彩治愈 / 深色专业 / 清新植物 / 极简黑白），附一句适用理由。
2. **设备形态**。竖屏手机 / 横屏平板 / 桌面 Web。
3. **主色**。给具体色值候选。

## 第 2 步 · DOM 结构约定（下游依赖，不可更改）

```html
<nav id="sidebar">
  <div class="group">业务分组名</div>
  <a data-s="<页面id>">N. 页面名</a>   <!-- 按用户旅程排序编号 -->
</nav>
<main id="main">
  <div class="phone" id="s-<页面id>"> ... </div>   <!-- 每页一个容器，id 前缀 s- -->
  <div class="notes" id="n-<页面id>"> 交互说明 </div>
</main>
<script>
  function go(s){...}   <!-- 全局跳转函数，页面内所有按钮用 onclick="go('目标id')" 串起真实业务流 -->
</script>
```

页面 id 沿用 PRD 页面清单里的 id。这套约定让下游截图脚本用 `go('{screen}')` + `#s-{screen}` 两个模板就能截完全部页面。

## 第 3 步 · 高保真的"真"

线框图和高保真的区别在内容密度，做到这些才配叫高保真：

- **真文案**。每个按钮、每条列表、每句提示都写真实业务文案，禁止 lorem 和"标题占位"。数字要合理（进度 48/300 而不是 0/0）。
- **真状态**。PRD 里的空状态、错误态、回归态各自成页。
- **人格化**。产品有 IP/吉祥物时让它在界面里说话（气泡、便签），演示视频里这是记忆点。
- **CSS 变量定义全套配色**，`:root{--主色:...}`，风格改起来一处生效。
- **每页交互说明**写三类条目：氛围（为什么这么设计）、规则（数值与边界）、埋点/验收。
- 单文件，无外部依赖，file:// 直接打开可用；插画用 emoji 或内联 SVG。

## 第 4 步 · 自查与交付

1. 逐条对照 PRD 页面清单，一页不缺；`go()` 跳转按用户旅程能一路点通。
2. 有浏览器工具（headless Chrome / playwright / Chrome MCP）就抽 2-3 页截图目检，文字溢出、布局塌了要修。
3. 交付 HTML 文件，附一句每组页面的浏览引导。
4. 主动告诉用户下一步可用 hifi-to-storyboard 把交互稿变成演示视频分镜表与分镜图。
